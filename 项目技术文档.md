# PlanMosaic 项目技术文档

## 项目概述

PlanMosaic 是一个创新的智能日程管理与词汇学习综合平台，采用多平台架构设计，为用户提供高效的时间管理和知识学习一体化解决方案。项目通过深度集成AI技术，实现了智能化的日程规划和个性化的学习体验。

## 核心功能模块

### 1. 智能日程管理 (PlanMosaic)

**主要功能特性**：
- **可视化时间轴**：支持日/周视图切换的直观时间管理界面
- **智能时间分配**：基于优先级和约束条件的自动时间段安排
- **任务进度追踪**：实时监控任务完成状态和进度百分比
- **里程碑标记**：重要事件和关键节点的突出显示系统
- **冲突检测机制**：自动识别并提示日程安排冲突
- **模板化管理**：支持常用日程模板的快速应用

**技术实现**：
- 基于`DaySchedule`数据模型的统一数据结构
- 响应式UI设计，完美适配不同屏幕尺寸
- 本地缓存与云端同步的双重数据保障机制

### 2. 词汇学习系统 (WordMosaic)

**核心学习功能**：
- **多专业词库**：集成大学英语四级、六级、考研、医学等专业词汇
- **智能学习算法**：基于艾宾浩斯遗忘曲线的自适应复习安排
- **多样化学习模式**：
  - 单词卡片学习
  - 拼写练习测试
  - 选择题评估
  - 上下文理解训练
- **学习进度分析**：详细的统计数据可视化和错误模式识别
- **AI辅助学习**：智能单词解释和个性化例句生成

**词库架构**：
```json
词库体系：
- college1.json: 大学英语四级词汇 (约3000词)
- college2.json: 大学英语六级词汇 (约5000词)
- college3.json: 考研英语核心词汇
- medical.json: 医学专业英语词汇
```

### 3. Mosa AI智能助手

**AI能力集成**：
- **自然语言交互**：支持日程和学习的自然语言指令
- **智能建议生成**：基于上下文和历史数据的个性化推荐
- **自动化任务处理**：智能创建、修改、优化日程安排
- **多轮对话记忆**：保持会话连续性，深度理解用户意图

**AI工具生态系统**（9大核心工具）：
1. `view_schedule` - 日程查看与分析
2. `add_schedule` - 智能日程添加
3. `modify_schedule` - 日程优化调整
4. `check_conflicts` - 冲突检测与解决
5. `manage_tasks` - 任务管理自动化
6. `manage_big_tasks` - 大型项目管理
7. `manage_templates` - 模板智能应用
8. `extract_profile` - 用户画像构建
9. `quantitative_analysis` - 数据量化分析

### 4. 深度规划模块

**战略规划能力**：
- **长期目标设定**：支持人生目标的系统性规划
- **量化决策支持**：数据驱动的优先级评估系统
- **价值分析工具**：多维度的价值评估框架
- **结构化输出**：可执行的详细行动计划生成

### 数据库设计

**Supabase表结构**：
```sql
-- 用户基础表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username TEXT UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 日程数据表（分区存储）
CREATE TABLE schedules (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    date DATE NOT NULL,
    data JSONB NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 词汇学习进度表
CREATE TABLE vocab_progress (
    user_id UUID REFERENCES users(id),
    book_id TEXT NOT NULL,
    word TEXT NOT NULL,
    status TEXT CHECK (status IN ('new', 'learning', 'reviewing', 'mastered')),
    next_review TIMESTAMP,
    stats JSONB,
    PRIMARY KEY (user_id, book_id, word)
);

-- 对话历史表
CREATE TABLE chat_history (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    session_id TEXT NOT NULL,
    messages JSONB NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
```
