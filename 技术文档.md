**preload.js 安全模型**

```javascript
// 通过 contextBridge 暴露安全API
contextBridge.exposeInMainWorld('electronAPI', {
    getScheduleData: () => ipcRenderer.invoke('get-schedule-data'),
    saveScheduleData: (data) => ipcRenderer.invoke('save-schedule-data', data),
    agentChatStream: (msg, history) => {...},
    // ... 其他接口
});
```

- 使用 `contextBridge` 实现进程隔离
- 渲染进程无直接Node.js访问权限
- 所有通信通过明确定义的IPC通道

    date: string;              // 日期 YYYY-MM-DD

    timeSlots: TimeSlot[];     // 时间段列表

// 时间段
interface TimeSlot {
    time: string;              // 时间范围 HH:MM-HH:MM

### 4.2 AI API集成

**双提供商架构**

```
AI Provider Interface
├── DeepSeek (默认)
│   ├── URL: https://api.deepseek.com/v1/chat/completions
│   ├── 快速模型: deepseek-chat
│   ├── 推理模型: deepseek-reasoner
│   └── 特性: Function Calling, Streaming, Reasoning
│
└── Qwen (备选)
    ├── URL: https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
    ├── 模型: qwen3.5-plus
    └── 特性: Function Calling, Streaming
```

**API调用流程**

```
1. 用户发送消息
      ↓
2. 构建系统提示词 (含日期上下文 + 用户画像)
      ↓
3. 意图检测 → 选择模型 (快速模型 or 推理模型)
      ↓
4. 发送API请求 (含工具定义)
      ↓
5. 接收响应
   ├── 普通文本回复 → 直接返回
   └── 工具调用请求 → 执行工具 → 收集结果 → 再次调用API (最多10轮)
      ↓
6. 生成Proposal (如有数据修改)
      ↓
7. 返回最终响应 (流式/非流式)
```

## 五、AI工具系统

### 5.1 工具定义 (ai-tools.js)

共定义**14个AI工具**，分为日程管理和深度规划两大类。

**日程管理工具 (9个)**

| 工具名 | 功能 | 关键参数 |

| `modify_schedule` | 修改日程 | operation, criteria, new_details |
| `check_conflicts` | 冲突检测 | date_range |
| `manage_tasks` | 任务管理 | action (add/view/complete/update/delete) |
| `manage_big_tasks` | 大型任务管理 | action, task_name, ddl, subtasks |
| `manage_courses` | 课程管理 | action, courses, weekday |
| `analyze` | 分析优化 | type (patterns/optimize/ddl/habits) |
| `manage_templates` | 模板管理 | action, template_name |

**深度规划工具 (5个)**

| 工具名 | 功能 |
|--------|------|
| `value_monetization` | 价值量化分析 |
| `roi_calculator` | ROI计算器 |
| `milestone_planner` | 里程碑规划 |
| `swot_analysis` | SWOT分析 |
| `decision_matrix` | 决策矩阵 |

### 5.2 工具执行引擎

**执行流程**

```
AI返回工具调用 (tool_calls)
      ↓
解析工具名称和参数
      ↓
旧名称兼容路由 (TOOL_ROUTING映射)
      ↓
参数适配转换
      ↓
执行工具逻辑 (读取/修改日程数据)
      ↓
生成执行结果文本
      ↓
返回结果给AI (作为tool message)
      ↓
AI决定是否继续调用工具或返回最终回复
```

**工具路由兼容表**（部分）

| 旧工具名 | 路由到 |
|----------|--------|
| list_schedules | view_schedule |
| search_schedules | view_schedule |
| add_recurring_schedule | add_schedule |
| batch_modify_schedules | modify_schedule |
| detect_schedule_conflicts | check_conflicts |
| add_task / complete_task | manage_tasks |
| add_big_task / break_down_big_task | manage_big_tasks |
| create_course_schedule | manage_courses |

### 5.3 Proposal确认机制

对涉及数据修改的操作，AI工具返回Proposal而非直接执行：

```
AI建议 → 生成Proposal → 前端展示 → 用户审批
                                    ├── 确认 → 执行变更 → 保存数据
                                    └── 拒绝 → 取消操作
```

支持的Proposal类型：
- 日程添加/修改/删除
- 任务添加/完成/修改/删除
- 大型任务管理
- 课程修改
- 批量操作

## 十、错误处理

### 10.1 AI API错误码处理

| HTTP状态码 | 含义 | 用户提示 |
|-----------|------|---------|
| 200 | 成功 | 正常返回 |
| 401 | 认证失败 | API密钥无效，请检查配置 |
| 402 | 余额不足 | API账户余额不足，请充值 |
| 403 | 权限不足 | 无访问权限 |
| 429 | 请求频率限制 | 请求过于频繁，请稍后再试 |
| - | 网络错误 | 检查网络连接 |
| - | DNS错误 | DNS解析失败 |
| - | SSL错误 | SSL证书验证失败 |
| - | 超时 | 请求超时，请增加超时时间 |

### 10.2 数据错误处理

- JSON解析失败：返回空数据结构
- 文件不存在：创建默认数据
- 写入失败：自动备份保障数据安全
- Unicode异常：Lone Surrogate清理机制

## 十三、技术债务与改进方向

### 13.1 当前技术债务

| 项目 | 描述 | 优先级 |
|------|------|--------|
| 单文件前端 | index.html包含所有前端代码，体积达352KB | 中 |

### 13.2 架构改进建议

1. **前端模块化**：将index.html拆分为独立的JS/CSS模块
2. **引入TypeScript**：增强类型安全，减少运行时错误
3. **添加测试框架**：Jest (桌面端) + JUnit (Android端)
4. **环境变量管理**：使用.env文件管理敏感配置
5. **CI/CD流水线**：自动化构建、测试、发布流程
6. **Room数据库**：Android端考虑引入Room替代纯JSON存储
7. **实时同步**：引入Supabase Realtime实现多端实时同步
